The permanent constraint

No matter how mature agentic automation gets, a human gate stays in place — to govern its work, to surveil its decisions, to intervene when it matters.

That's not a gap left to close as agents get smarter. It's a constraint to design for. The question worth answering isn't "how do we remove the human" — it's "how is that gate actually executed today, and what does that choice cost us."

How the gate is executed today

Welded into the agent. That's the whole problem.

Today's gate is a raw interrupt() and a bespoke Slack message, written once, inside a single agent's code. It works — until it needs to change, or be seen, or be trusted.

What welding it in costs you

Not a bug. A ceiling.

None of this shows up in a demo. It shows up the first time you try to change, audit, or trust how your agents hand off to people.

✕

It can't be governed

There's no single place to see how, or how well, every agent's human hand-off actually behaves.

✕

It can't be changed without redeploying

Who gets asked, and how, is compiled into the agent — evolving it means shipping code, not flipping a setting.

✕

It can't learn

Every past human decision is a Slack message, not a record — there's no substrate for the gate to get smarter over time.

✕

It can't be trusted at scale

N agents means N one-off implementations of the same checkpoint — no consistency, no shared guarantee, no audit trail.

The integration

What the shift looks like in your code.

Two lines of your own logic. Everything after the interrupt belongs to the gateway now.

1

Decorate your HITL node

@hg.hitl_node binds the current LangGraph state and config so the SDK can build context automatically.

2

Raise the interrupt

hg.raise_interrupt() registers with the gateway and suspends the graph — the typed response comes back on resume.

import hgateway_sdk as hg
from hgateway_sdk import BinaryApprovalContent, RunFeatures, Routing, Recipients

@hg.hitl_node
def review_node(state):
    content = BinaryApprovalContent(prompt=f"Approve {state['summary']}?")

    resp = hg.raise_interrupt(
        "approve-action", content,
        features=RunFeatures(routing=Routing(primary=Recipients(channels=["#ops"]))),
        fallback={"decision": "reject"},
    )
    return {"decision": resp["decision"]}
